Vuex-购物车实战总结

概述

  • 是什么

    vuex是一个vue的状态管理工具,状态就是数据

  • 使用场景

    • 某个状态在很多组件来使用(个人信息)
    • 多个组件共同维护一份数据(购物车)

  • 优势

    • 共同维护一份数据,数据集中化管理
    • 响应式变化
    • 操作简单(vuex提供了一些辅助函数)

  • 图示

state 状态

    明确如何给仓库提供数据,如何使用仓库的数据

mutaions

  • 目标: 明确vuex同样遵循单向数据流,组件中不能直接修改仓库的数据

  • 通过strict: true可以开启严格模式

  • 辅助函数出现在methods中

actions

  • 目标: 明确actions的基本用法,处理异步操作

module

  • 由于vuex使用单一状态树,应用的所有状态会集中到一个比较大的对象，当应用变得非常复杂时,store对象就有可能变得相当臃肿

  • 模块拆分:store/modules/xxx.js

  • 在index.js中的modules中使用

    • user

    • setting

    • index.js

      • 使用模块中的数据

        1. 直接通过模块名访问 $store.state.模块名.xxx

        2. 通过mapState映射

          子模块的映射mapState('模块名',['xxx'])-需要开启命名空间(namespaced:true)
