GIT Rebase

  • 概念

    git rebase 的目标是创建一个更线性、清晰的提交历史

    git rebase master 用于将当前分支的(在bugFix)提交整合到 master 分支上，创建一个更为整洁的提交历史。这个操作通常在你希望保持提交历史清晰、避免不必要的合并提交的情况下使用。

  • 目标

  • 基点

  • 新建并切换到该分支

  • 在bugFix上提交一次

  • 切回到main分支

  • 在main分支上提交一次

  • 切回到bugFix

  • rebase到main上

分离头指针

  • 分离头指针模型

    在Git中，”分离头指针”是指你正在查看历史提交，而不是在分支上进行工作。这种情况下，你不能进行提交，因为没有分支指向你的新提交。如果你想保存你的更改，你应该创建一个新的分支。

致命：拒绝合并不相关的历史记录”Git 错误

  • 当两个不相关的项目（即不知道彼此存在且提交历史记录不匹配的项目）合并时，会出现“致命：拒绝合并不相关的历史记录” Git

    • 考虑以下两种引发此错误的情况：
      • 您克隆了一个项目，但不知何故，.git目录被删除或损坏。这会导致 Git 不知道您的本地历史记录，因此当您尝试推送到远程存储库或从远程存储库拉取时，会导致它抛出此错误。
      • 您已经创建了一个新存储库，并向其中添加了一些提交，现在您正尝试从已经有自己的一些提交的远程存储库中提取数据。在这种情况下，Git 也会抛出错误，因为它不知道这两个项目是如何关联的。

  • 解决方案

    通过切换允许无关历史记录开关可以解决该错误。在git pullorgit merge命令后添加以下标记
