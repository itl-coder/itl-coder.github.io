  3. 事务的四大特性是（原子性）（一致性）（隔离性）（持久性）

    • 原子性：一个事务中,操作要么全部执行，要么都不执行
    • 一致性：在事务的执行前后,数据在逻辑上保持一致
    • 隔离性: 多个事务并发操作相同的数据时,一个事务对其他事务的影响,具体如何影响,取决于隔离级别
    • 持久性: 一个事务一旦结束(提交或回滚),对数据库中的数据的影响会一直保存下去,任何设备故障都不会影响存储的数据

  4. Spring实例化bean的3种方式是

    • 构造器实例化

    • 静态工厂方法实例化

    • 实例工厂方法实例化

  5. Git的三大区域是

    • 工作区： 工作区是用户进行代码编辑和修改的地方，也是Git管理的最小单元。工作区中的所有文件都可以被Git跟踪和管理
    • 暂存区：暂存区是位于工作区和版本库之间的缓冲区。用户可以将工作区中的改动暂存到暂存区，然后再提交到版本库。暂存区可以帮助用户更好地组织和管理代码提交
    • 版本库：版本库是Git用来存储代码历史记录的地方。版本库中的所有文件都是只读的，用户无法直接修改版本库中的文件

二．简答

  1. 简述@Autowired和@Resources的区别
  • @Autowired是 spring 提供的，@Resource 是 jdk 内置的
  • @Autowired 是优先按照类型查找
    • 找到一个就装配
    • 找不到，就看 required 属性的取值，true 就报错，false 就不报错，而是注入null
    • 找到多个：就按照名称找，找到就注入，找不到就报错
  • @Resource 优先按照名称找
    • 找到一个就装配
    • 找不到就报错
    • 找到多个也报错

  2. 脏读、幻读、不可重复读分别是什么？

    • 脏读

      当一个事务读取到另一个事务尚未提交的数据时，发生了脏读，如果另一个事务回滚，读取到的数据就是无效的

    • 幻读

      当一个事务在读取某个范围内的数据时，另一个事务在该范围内插入了新的数据，导致第一个事务的读取结果发生了变化，就发生了幻读

    • 不可重复读

      当一个事务在读取某个数据后，另一个事务修改了该数据，导致第一个事务再次读取时得到不同的结果，就发生了不可重复读

  3. 简述SpringMVC的工作流程？

    1. 用户发起请求 login.do

    2. DispatcherServlet 接受请求 login.do, 把请求转交给处理器映射器

      处理器映射器: springmvc 框架中的一种对象，框架把实现了 HandlerMapping 接口的类叫做映射器

      处理器映射器的作用：根据请求，从springmvc容器对象中获取处理器对象

    3. DispatcherServlet 把 2 中的 HandlerExecutionChain 中的处理器对象交给了处理器适配器对象

      处理器适配器: springmvc框架中的对象，需要实现 HandlerAdapter 接口

      处理器适配器的作用：执行・处理器方法 (调用 Controller.doLogin(),得到返回值 ModelAndView)

    4. DispatcherServlet 把 3 中获取的 ModelAndView 交给了视图解析器对象

      视图解析器: springmvc 中的对象，需要实现 ViewResoler 接口

      视图解析器的作用：组成视图完整路径，使用前缀，后缀，并创建 view 对象

      View 是一个接口，表示视图的，在框架中 jsp，html 不是 string 表示，而是使用 View 和他的实现类表示视图

      InternalResourceView: 视图类，表示 jsp 文件，视图解析器会创建 InternalResourceView 类对象，这个对象的里面，有一个属性 url=/WEB-INF/view/show.jsp

    5. DispatcherServlet 把 4 步骤中创建的 View 对象获取到，调用 View 类自己的方法，把 Model 数据放入到 request 作用域

  1. 简述SpringBoot自动配置的原理？

    • 在Spring Boot项目中的引导类上有一个注解@SpringBootApplication,这个注解是对三个注解的封装,分别是
      • @SpringBootConfiguration
      • @EnableAutoConfiguration
      • @ComponentScan
    • 其中@EnableAutoConfiguration是实现自动化配置的核心注解,该注解通过@Import注解导入对应的配置选择器，内部就是读取了该项目和该项目引用的jar包的classpath路径下META-INF/spring.factories文件中的所配置的类的全类名,在这些配置类中所定义的Bean会根据条件注解所指定的条件来决定是否需要将其导入到Spring容器中
    • 条件判断会有向@ConditionalOnClass这样的注解,判断是否有对应的class文件,如果有则加载该类,把这个配置类的所有Bean放入spring容器中使用

  2. 你知道的SpringBoot注解？至少写5个

    1. @SpringBootApplication:

    • 是 Spring Boot 的核心注解，用于开启 Spring Boot 自动配置。
    • 该注解组合了 @Configuration, @EnableAutoConfiguration 和 @ComponentScan 三个注解。
    • 可以通过 @SpringBootApplication(exclude = {…}) 来排除自动配置的类。

    2. @EnableAutoConfiguration:

    • 启用 Spring Boot 的自动配置功能，会自动根据类路径中的 jar 包来配置 Spring 容器。
    • 可以通过 @EnableAutoConfiguration(exclude = {…}) 来排除自动配置的类。

    3. @ComponentScan:

    • 用于扫描指定包路径下的类，并将其注册到 Spring 容器中。
    • 可以通过 @ComponentScan(basePackages = {…}) 来指定扫描的包路径。

    4. @ConfigurationProperties:

    • 用于将外部配置文件中的属性绑定到 Java bean 的属性上。
    • 可以通过 @ConfigurationProperties(prefix = "…") 来指定属性的前缀。

    5. @ConditionalOnProperty:

    • 根据指定的条件判断是否加载该类或方法。
    • 可以通过 @ConditionalOnProperty(name = "…", havingValue = "…") 来指定条件。

    6. @ConditionalOnBean:

    • 根据指定的 bean 是否存在来判断是否加载该类或方法。
    • 可以通过 @ConditionalOnBean(type = {…}) 来指定 bean 的类型。

    7. @ConditionalOnClass:

    • 根据指定的类是否存在来判断是否加载该类或方法。
    • 可以通过 @ConditionalOnClass(name = "…") 来指定类的名称。

    8. @ConditionalOnMissingBean:

    • 根据指定的 bean 是否不存在来判断是否加载该类或方法。
    • 可以通过 @ConditionalOnMissingBean(type = {…}) 来指定 bean 的类型。

  1. JDK动态代理和CGLIB动态代理的区别？

    在java的动态代理机制中，有两个重要的类或接口，一个是 InvocationHandler(Interface)、另一个则是 Proxy(Class)，这一个类和接口是实现我们动态代理所必须用到的。

    每一个动态代理类都必须要实现InvocationHandler这个接口，并且每个代理类的实例都关联了一个handler，当我们通过代理对象调用一个方法的时候，这个方法的调用就会被转发为由InvocationHandler这个接口的 invoke 方法来进行调用。

    InvocationHandler这个接口的唯一一个方法 invoke 方法：

    个方法一共接受三个参数，那么这三个参数分别代表如下：

    • proxy:指代JDK动态生成的最终代理对象
    • method:指代的是我们所要调用真实对象的某个方法的Method对象
    • args:指代的是调用真实对象某个方法时接受的参数

    Proxy这个类的作用就是用来动态创建一个代理对象的类，它提供了许多的方法，但是我们用的最多的就是newProxyInstance 这个方法

    这个方法的作用就是得到一个动态的代理对象，其接收三个参数，我们来看看这三个参数所代表的含义：

    • loader：ClassLoader对象，定义了由哪个ClassLoader来对生成的代理对象进行加载，即代理类的类加载器。
    • interfaces：Interface对象的数组，表示的是我将要给我需要代理的对象提供一组什么接口，如果我提供了一组接口给它，那么这个代理对象就宣称实现了该接口(多态)，这样我就能调用这组接口中的方法了。
    • Handler：InvocationHandler对象，表示的是当我这个动态代理对象在调用方法的时候，会关联到哪一个InvocationHandler对象上。
