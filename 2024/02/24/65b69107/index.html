Nginx-负载均衡的方式

  • 轮询（Round Robin）：
    • 轮询策略按顺序将每个新的请求依次分配给服务器列表中的下一个服务器。这是最简单的负载均衡算法之一，适用于每台服务器具有相似处理能力和负载的情况。
  • 最少连接（Least Connections）：
    • 最少连接策略将请求分配给当前具有最少连接数的服务器。这样可以确保将请求发送到当前最空闲的服务器上，有助于避免过载。
  • IP 哈希（IP Hash）：
    • IP 哈希策略根据客户端 IP 地址将请求分配给特定的服务器。这样可以确保相同的客户端请求总是被发送到同一台服务器上，有助于保持一些状态相关的数据。
  • 加权轮询（Weighted Round Robin）：
    • 加权轮询策略与轮询类似，但是为每个服务器分配了一个权重，权重越高的服务器会获得更多的请求。这样可以根据服务器的性能和资源分配不同的负载。
  • 加权最少连接（Weighted Least Connections）：
    • 加权最少连接策略结合了最少连接和加权轮询策略的思想，根据服务器的权重和当前连接数来选择服务器。这样可以将负载更加均匀地分配到服务器上。
  • 随机（Random）：
    • 随机策略通过随机选择一个服务器来处理每个请求。虽然简单，但是在某些情况下可能会导致负载不均匀。
  • URL 哈希（URL Hash）：
    • URL 哈希策略根据请求的 URL 将请求分配给特定的服务器。这样可以确保相同的 URL 总是被发送到同一台服务器上，有助于缓存和提高命中率。
